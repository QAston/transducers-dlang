void  flush();

ProcessDecorators which do buffering need to override this  flush method. This method allows to process remaining input and feed it to the wrapped process by calling step, just like step methods do. After all processing is done the decorator should call process on wrapped process.

bool  isTerminatedEarly(Process)(ref Process process);

Returns true when process is marked for early termination.

Transducible context should check for this after every step, and if true stop calling step, call flush and finish.

void  markTerminatedEarly(Process)(ref Process process);

Marks underlying process for early termination.

Use inside ProcessDecorator.step() method prevent the transducible context from feeding the process more input. When marked, no new input can be fed to the process using step, outside of flush method.
